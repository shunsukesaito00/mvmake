## 実装済み機能の完成度見直し（2026-07-09）

v1.5.0 リリース後の実利用報告（数 MB 動画取り込みで UI フリーズ）を起点に、実装済み 69 項目を再評価した。

### 評価区分

| 区分 | 意味 |
|------|------|
| **本番品質** | 実利用で問題になりにくい。E2E または同等の手動確認済み |
| **MVP 要磨き込み** | 機能は動くが UX・エッジケース・パフォーマンスに改善余地 |
| **既知不具合** | 実利用で再現確認済み。修正計画に組み込み済み |

### 本番品質（コア編集フロー）

タイムライン操作（移動・トリム・分割・複製・Undo/Redo）、テキスト編集、トランジション適用、Ken Burns、BGM ダッキング、書き出し（WebCodecs）、プロジェクト保存/復元、`.fable` 入出力、オンボーディング・サンプルプロジェクト、キーボードショートカット群。

### MVP 要磨き込み（機能はあるが品質向上余地）

| 機能 | 課題 |
|------|------|
| 音量キーフレーム + タイムライン UI | 操作感・細かいスナップ |
| カラールック / 映像フェード | プリセット数・プレビュー精度 |
| 章マーカー一括 ZIP 書き出し | 長尺・多数章での進捗・メモリ |
| Google Fonts | フォント読み込み待ちの体感 |
| ナレーション録音 | デバイス権限エラー時の案内 |
| SRT 字幕インポート / エクスポート | エンコーディング・長文の折り返し |
| メディア差し替え | 映像↔画像のクロス差し替え未対応 | **v1.7.0 で対応** |
| 大容量メディア対策 | 警告のみで実フリーズが発生していた（→ v1.5.1 で修正） |

### 既知不具合 → 修正状況

| 不具合 | 深刻度 | 原因 | 対策 | 状態 |
|--------|--------|------|------|------|
| **数 MB 動画取り込みで UI フリーズ** | 高 | `onloadeddata` による全フレームデコード待ち、サムネ・波形をインポート完了まで同期処理、ファイル間 yield なし | メタデータのみ先に取り込み、サムネは `loadedmetadata`+`seeked`、波形は先頭 5MB のみ、バックグラウンド enrich | **v1.5.1 修正済・リリース** |
| 大容量 IndexedDB 自動保存の遅延 | 中 | Blob 丸ごと保存で数百 MB 超が重い | 進捗表示・容量使用率表示・詳細エラー通知 | **v1.6.1 改善済** |
| 長時間書き出しの進捗体感 | 低 | フレーム単位進捗のみ | 残り時間推定・キャンセル後の状態整理 | **v1.8.0 改善済** |

### 本番スモーク E2E (`npm run test:e2e:prod` / deploy smoke)

GitHub Pages 公開環境に対して `e2e/basic.spec.ts` の 3 シナリオを実行する。
