# 有名動画編集アプリ vs FABLE 機能比較

最終更新: 2026-07-13（v2.6.35 本番スモーク E2E 687→690拡充・動画/画像TFスケール数値入力/動画TFグラフエディター3件）

## プロダクト目標

**結婚式ムービーが作りやすいブラウザ完結型エディタ**を目指す。CapCut / **Premiere Pro クラスのタイムライン編集・操作性**を婚礼制作フロー（構成→素材投入→テロップ/BGM→仕上げ→章ごと書き出し）に最適化する。

| 観点 | 目標 | v1.68.0 時点の評価 |
|------|------|-------------------|
| **婚礼ワークフロー** | テンプレ・章・写真ガイド・SRT・章 ZIP など一連が途切れない | **達成** |
| **Premiere コア編集** | マルチトラック・トリム/分割・キーフレーム・In/Out・マーカー | **おおむね同等** |
| **Premiere 操作性** | J/K/L・リップル編集・タイムライン上キーフレーム操作 | **おおむね同等** — スリップ/スライド編集追加 |
| **Premiere 仕上げ** | Lumetri 色調・多数トランジション・速度ランプ・調整レイヤー | **改善中** — LUT+Canvasプレビュー+トーン/RGBカーブ+HSL・ルック22種+ピクセルグレードミニプレビュー・トランジション29種・速度KF・調整レイヤー |
| **総合（婚礼用途中心）** | Premiere で婚礼ムービーを作る体験に近づける | **約 86〜90%**（機能数は十分、**仕上げの一貫性**が次の伸びしろ） |

> 「同等」とは婚礼ムービー制作で**日常的に使う機能・操作**が Premiere と同程度使えること。AI 自動編集・マルチカム・共同編集などスコープ外は対象外（[スコープ外](#スコープ外)参照）。

| 不具合 | 深刻度 | 原因 | 対策 | 状態 |

| **動画クリップ再生中に停止できない** | **高** | `usePlayback` が PreviewPanel / TimelinePanel / MarkerInspector の **3 箇所で独立インスタンス化**され、各所が `requestAnimationFrame` ループを起動。`tick` 終了時に `playingRef` を見ず次フレームを予約するため、一時停止直後の stale tick が `syncVideosForPlayback(..., playing: true)` で `video.play()` を再開しうる | `PlaybackProvider` で **1 回だけ**マウント。`tick` 先頭で `playingRef` ガード。`pauseAllVideos` で全 `<video>` を確実に `pause` | **v1.69.0 修正済** |
| **動画プレビューがカクつく** | **高** | 時刻マスターが `audioEngine` と `HTMLVideoElement.play()` の **二重同期**。`renderFrame` / `syncVideosForPlayback` が毎フレーム `video.currentTime` を強制シーク。PreviewPanel の描画 RAF と `usePlayback` の tick RAF が並走 | audioEngine **単一クロック**（再生中はシークのみ・`video.play()` 廃止）。描画ループを `subscribeFrame` で一本化。1080p 実測 FPS は Q7 で継続 | **v1.69.0 初回対応** |
